# FlagManager 低保真原型

## 1. 原型目标

本原型用于明确 FlagManager 微信小程序 MVP 阶段的核心页面结构、主要入口和用户操作路径。

当前重点不是视觉风格，而是确认：

- 用户如何新建长期 Flag。
- 用户如何查看长期目标进度。
- 用户如何进入阶段计划。
- 用户如何完成打卡。
- 用户如何查看统计数据。

## 2. 页面总览

### 2.1 TabBar

MVP 阶段建议使用 3 个底部导航：

- 首页
- 统计
- 我的

### 2.2 核心页面

- 首页
- 新建 Flag 页
- Flag 详情页
- 新建阶段页
- 阶段详情页
- 打卡页
- 统计页
- 我的页

## 3. 页面跳转关系

```text
首页
├─ 点击右上角 +
│  └─ 新建 Flag 页
├─ 点击 Flag 卡片
│  └─ Flag 详情页
├─ 点击今日待打卡
│  └─ 打卡页
└─ 点击底部统计
   └─ 统计页

Flag 详情页
├─ 点击添加阶段
│  └─ 新建阶段页
├─ 点击阶段卡片
│  └─ 阶段详情页
└─ 点击打卡
   └─ 打卡页

阶段详情页
└─ 点击打卡
   └─ 打卡页
```

## 4. 首页

首页承担两个任务：

- 展示用户当前最重要的长期 Flag。
- 提供新建 Flag 和快速打卡入口。

```text
┌────────────────────────┐
│ FlagManager          + │
│ 把立下的 Flag 兑现掉  │
├────────────────────────┤
│ 今日概览               │
│ 连续打卡 12 天         │
│ 本月打卡 18 次         │
├────────────────────────┤
│ 今日待打卡             │
│ ┌────────────────────┐ │
│ │ 考研英语           │ │
│ │ 阶段：单词强化     │ │
│ │ 目标：今天背50词   │ │
│ │              去打卡 │ │
│ └────────────────────┘ │
├────────────────────────┤
│ 进行中的 Flag          │
│ ┌────────────────────┐ │
│ │ 完成考研英语       │ │
│ │ 2026.02 - 2026.08  │ │
│ │ 进度 45%           │ │
│ │ 当前阶段：单词强化 │ │
│ └────────────────────┘ │
│ ┌────────────────────┐ │
│ │ 坚持健身三个月     │ │
│ │ 进度 30%           │ │
│ │ 当前阶段：基础适应 │ │

├────────────────────────┤
│ 首页      统计      我的│
```

### 4.1 首页交互

- 点击右上角 `+`：进入新建 Flag 页。
- 点击 `[ + 新建 Flag ]`：进入新建 Flag 页。
- 点击 Flag 卡片：进入 Flag 详情页。
- 点击 `去打卡`：进入打卡页。

## 5. 新建 Flag 页

新建 Flag 页用于创建长期目标，不放太多复杂字段，避免用户刚开始就有压力。

```text
┌────────────────────────┐
│ < 新建 Flag             │
├────────────────────────┤
│ Flag 名称               │
│ ┌────────────────────┐ │
│ │ 例如：完成考研英语 │ │
│ └────────────────────┘ │
├────────────────────────┤
│ Flag 描述               │
│ ┌────────────────────┐ │
│ │ 写下你为什么要做...│ │
│ └────────────────────┘ │
├────────────────────────┤
│ 分类                    │
│ [学习] [健身] [阅读]    │
│ [项目] [其他]           │
├────────────────────────┤
│ 开始日期                │
│ 2026-06-09              │
├────────────────────────┤
│ 目标完成日期            │
│ 2026-12-31              │
├────────────────────────┤
│ [创建 Flag]             │
└────────────────────────┘
```

### 5.1 新建 Flag 后

创建成功后进入 Flag 详情页，并提示用户添加第一个阶段计划。

### 6.1 Flag 详情页交互

- 点击 `···`：编辑、暂停、完成、放弃 Flag。
- 点击阶段卡片：进入阶段详情页。
- 点击 `[ + 添加阶段 ]`：进入新建阶段页。
- 点击 `[打卡]`：进入打卡页，默认选择当前进行中的阶段。

打卡页要尽量轻量，降低用户记录成本。

```text
┌────────────────────────┐
│ < 今日打卡             │
├────────────────────────┤
│ Flag                   │
│ 完成考研英语           │
├────────────────────────┤
│ 阶段                   │
│ 单词强化          切换 │
├────────────────────────┤

│ ┌────────────────────┐ │
│ │ 今天背了50个单词， │ │
│ │ 复习了昨天内容...  │ │

├────────────────────────┤
│ 添加图片               │

├────────────────────────┤
```

### 9.1 打卡后

提交成功后返回来源页面：

- 从首页进入：返回首页。
- 从 Flag 详情进入：返回 Flag 详情页。
- 从阶段详情进入：返回阶段详情页。

## 12. MVP 首屏重点

首页首屏应优先出现：

- 产品名称
- 新建 Flag 入口
- 今日待打卡
- 进行中的 Flag

不建议首页一开始放太多统计图表，否则会削弱“继续推进长期目标”的核心行为。

## 13. 后续可视化方向

低保真确认后，可以继续做：

- 中保真页面结构
- 视觉风格方案
- uni-app 页面文件
- 微信小程序可运行原型

建议下一步先确认首页和 Flag 详情页，因为它们决定整个产品的核心体验。
